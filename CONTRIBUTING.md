# Contributing to Awesome Generative AI

Thank you for your interest in contributing! This list thrives because of community contributions like yours. 🙌

## How to Contribute

### Suggesting a Resource

1. **Check for duplicates** — Search the README to make sure the resource isn't already listed.
2. **Fork this repository** and create a new branch.
3. **Add your resource** in the appropriate section, following the format below.
4. **Submit a Pull Request** with a clear description of what you're adding and why.

### Resource Format

For tools/frameworks:
```
- **[Tool Name](https://link.to/tool)** — Brief description (1-2 sentences). `Language/Platform`
```

For table entries, follow the existing column structure in that section.

### Quality Guidelines

We aim to maintain a high-quality, curated list. Resources should meet these criteria:

- ✅ **Actively maintained** — Updated within the last 6 months
- ✅ **Relevant** — Directly related to Generative AI
- ✅ **Useful** — Provides real value to developers/researchers
- ✅ **Accessible** — Has documentation or clear instructions
- ✅ **Not spam** — No self-promotional content without substantial value

### What We're Looking For

- 🆕 New tools, frameworks, or models
- 📝 Improved descriptions or corrections
- 🏷️ Better categorization suggestions
- 🌍 Resources in languages other than English
- 📊 Updated comparison data

### What We'll Decline

- ❌ Tools/resources that are abandoned or unmaintained
- ❌ Paid-only resources without a free tier (unless they're industry-standard)
- ❌ Low-quality or duplicate resources
- ❌ Self-promotion without genuine community value

## Pull Request Process

1. Ensure your contribution follows the formatting guidelines above.
2. Provide a brief explanation of why this resource is awesome.
3. One resource per PR is preferred (but adding related resources together is fine).
4. PRs are reviewed within 48 hours.

## Reporting Issues

Found a broken link? Outdated information? Please [open an issue](https://github.com/prateekpatil0805-ui/newfolder/issues/new) with:
- The section and resource name
- What's wrong (broken link, outdated info, etc.)
- Suggested fix (if any)

## Code of Conduct

Please note that this project follows a [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you agree to abide by its terms.
